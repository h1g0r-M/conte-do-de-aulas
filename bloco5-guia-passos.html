HTML & CSS do Zero

Guia de passo a passo — aula estruturada em 4 blocos incrementais com tema no Sistema de Chamados

Somente HTML e CSS  ·  Sem frameworks  ·  Sem JavaScript

Trilha da aula — 4 blocos + este guia

1 Estrutura HTML Pura Arquivo: bloco1-estrutura.html
  1. Crie o arquivo e declare o DOCTYPE Abra um editor de texto, salve como .html e escreva <!DOCTYPE html> na primeira linha. Isso diz ao navegador qual versão do HTML está sendo usada (HTML5).
  2. Defina a estrutura raiz Adicione <html lang="pt-BR">, depois <head> com <meta charset="UTF-8"> e <title>. O lang ajuda leitores de tela e mecanismos de busca.
  3. Crie as seções semânticas no <body> Use <header>, <nav>, <main> e <footer>. Semântica = significado: cada tag descreve o papel do conteúdo, não só sua aparência.
  4. Escreva listas e parágrafos <ul> para itens sem ordem de importância, <ol> para passos sequenciais. Observe como o navegador aplica estilos padrão sem nenhum CSS.
  5. Monte o formulário de chamado Adicione <form> com os campos: input type="text", type="email", <select>, <fieldset> com radios de departamento, <textarea> e type="checkbox". Sempre vincule label ao campo via atributos for e id iguais.

2 Introdução ao CSS Arquivo: bloco2-css-basico.html
  1. Adicione a tag <style> no <head> Todo o CSS desta aula fica dentro de <style>...</style>, sem arquivo externo. Isso mantém tudo em um único arquivo para facilitar o aprendizado.
  2. Aplique estilos ao <body> — tipografia e cor de fundo Use font-family com uma pilha de fontes, background-color e color. O seletor de elemento aplica o estilo a todas as tags daquele tipo.
  3. Entenda o Box Model Cada elemento é uma caixa: content → padding → border → margin. Use box-sizing: border-box no início do CSS para que padding e border não somem à largura declarada.
  4. Estilize o header, nav e footer Aplique background-color, padding e color. Use o seletor descendente (nav a) para atingir apenas os links dentro da navegação.
  5. Centralize o main com width + margin auto width: 860px define a largura fixa (versão desktop). margin: 32px auto centraliza horizontalmente: o navegador distribui o espaço restante igualmente nos dois lados.
  6. Formate os campos do formulário display: block no label empurra o input para a linha seguinte. width: 100% nos inputs os faz ocupar toda a largura disponível. Use :focus para destacar o campo ativo.
  7. Use pseudo-classes :hover e :focus São estados do elemento. :hover se ativa quando o mouse passa por cima; :focus quando o campo está selecionado. Indicam interação sem JavaScript.

3 Layout com Display e Position Arquivo: bloco3-layout-position.html
  1. Crie cards de status com display: inline-block Antes do Flexbox, a técnica clássica para colunas era display: inline-block + width em porcentagem. Use font-size: 0 no elemento pai para eliminar o espaço em branco entre os elementos inline.
  2. Entenda position: relative e absolute relative: o elemento permanece no fluxo, mas vira referência para filhos absolute. absolute: o elemento sai do fluxo e se posiciona em relação ao seu pai posicionado mais próximo.
  3. Use pseudo-elementos ::before e ::after São conteúdo gerado pelo CSS — não existem no HTML. Precisam de content: "" para aparecer. Usados aqui para a faixa colorida no topo de cada card, sem criar elementos extras no HTML.
  4. Aplique position: fixed ao botão de notificações fixed posiciona o elemento em relação à janela do navegador, não ao documento. Ele permanece visível mesmo ao rolar a página. Controle a sobreposição com z-index.
  5. Monte a tabela de chamados recentes Use <table>, <thead>, <tbody>, <tr>, <th> e <td>. Adicione border-collapse: collapse para unir as bordas duplas entre células.
  6. Divida o formulário em duas colunas Crie uma div.form-row com font-size: 0 e dentro coloque div.form-col com display: inline-block; width: 48%. Use vertical-align: top para alinhar pelo topo quando as alturas diferirem.

4 CSS Responsivo com Media Queries Arquivo: bloco4-responsivo.html
  1. Adicione a meta viewport <meta name="viewport" content="width=device-width, initial-scale=1.0"> instrui o navegador mobile a usar a largura real da tela como base de cálculo. Sem ela, o celular finge ter 980px e escala a página.
  2. Substitua width fixo por max-width + width: % max-width: 940px; width: 95% no main: em telas grandes limita em 940px; em telas menores usa 95% da largura disponível. É mais flexível que um valor fixo em px.
  3. Envolva a tabela em um div com overflow-x: auto Em telas estreitas, tabelas largas quebram o layout. O wrapper com overflow-x: auto cria uma barra de rolagem horizontal só na tabela, sem afetar o resto da página.
  4. Escreva a primeira media query — tablet (768px) @media (max-width: 768px) { ... }: os estilos dentro das chaves se aplicam somente quando a tela tiver até 768px. Reorganize os cards de 4 para 2 por linha e empilhe as colunas do formulário.
  5. Escreva a segunda media query — mobile (480px) @media (max-width: 480px) { ... }: adapta para celular. Transforme os links do nav em blocos com display: block para ficarem empilhados. Reduza paddings e aumente o tamanho dos campos para facilitar o toque.
  6. Teste redimensionando a janela Abra o arquivo no navegador e arraste a borda da janela. Observe a página se reorganizar nos dois pontos de quebra. Use as Ferramentas do Desenvolvedor (F12 → ícone de celular) para simular dispositivos específicos.
