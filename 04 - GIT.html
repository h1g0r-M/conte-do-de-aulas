Git Local: Setup e Primeiro Commit

1

Configuração Inicial

Defina sua identidade (nome e e-mail) para que seus commits sejam autorados corretamente no histórico.

2

Inicializar Repositório

Crie a pasta do projeto e o subdiretório oculto .git que rastreará todas as mudanças.

3

Ciclo de Vida Básico

Verifique o status, adicione arquivos à staging area e consolide com o commit.

4

Primeiro Commit Estruturado

Use o padrão Conventional Commits para mensagens claras e rastreáveis desde o início.

bash — 80x24
# 1. Configurar identidade global
$git config --global user.name "Dev Responsável"
$git config --global user.email "dev@empresa.com"
# 2. Criar pasta e iniciar git
$mkdir sistema-solicitacao && cd sistema-solicitacao
~/sistema-solicitacao $git init
Initialized empty Git repository in .../.git/
# 3. Criar estrutura base e verificar
~/sistema-solicitacao (main) $touch README.md .gitignore
~/sistema-solicitacao (main) $mkdir src docs .github
~/sistema-solicitacao (main) $git status
Untracked files: .gitignore, README.md, docs/, src/
# 4. Adicionar tudo e realizar commit estruturado
~/sistema-solicitacao (main) $git add .
~/sistema-solicitacao (main) $git commit -m "feat: inicializa estrutura do projeto"

[main (root-commit) a1b2c3d] feat: inicializa estrutura do projeto

4 files changed, 12 insertions(+)

Use "git log --oneline" para verificar o histórico após o commit.
