Versionamento com Git — Conceitos‑chave

v1.0.0-git-basics

Por que versionar?

  • Histórico Completo: Linha do tempo de todas as alterações (quem, quando, o quê).

  • Segurança: Facilidade de reverter erros (rollback) sem perder código.

  • Colaboração: Trabalho simultâneo no mesmo projeto sem sobrescrever arquivos.

Conceitos Fundamentais

  • Repositório (.git): Onde o Git armazena os metadados e o banco de dados de objetos do seu projeto.

  • Working Directory: Sua pasta de trabalho atual onde você edita os arquivos.

  • Staging Area (Index): Área de preparação onde você escolhe o que entrará no próximo commit.

  • Commit: Um "snapshot" (foto) do projeto em um determinado momento.

Fluxo Básico de Vida

1. Editar
Alterar arquivos no Working Directory
git add
2. Preparar
Mover alterações para Staging Area
git commit
3. Gravar
Salvar snapshot no Repositório Local

Boas Práticas

  • Commits Atômicos: Faça commits pequenos focados em uma única tarefa ou correção.

  • Mensagens Claras: Use imperativo ("Adiciona..." e não "Adicionei"). Siga Conventional Commits.

  • Branches: Nunca commite direto na branch main/ master.
